ROOT CAUSE

Old code (lines 67-72):

        if ($this->is_perubahan) {
            return 'melakukan revisi/perbaikan';
        }
        return 'melakukan penilaian mandiri';

Bug: `is_perubahan` flag means "Penilaian row already existed before this action",
NOT "this is a revision action". So:
  - Upload pertama  → is_perubahan=false → returns 'melakukan penilaian mandiri' (WRONG, should be upload)
  - Scoring pertama → is_perubahan=true (row exists from upload step) → returns 'melakukan revisi/perbaikan' (WRONG)

FIX STRATEGY
------------
Distinguish action type via `tingkatan_nilai_id`:
  - tingkatan_nilai_id !== null → SCORING action
  - tingkatan_nilai_id === null → UPLOAD action

Delete actions detected via EXACT MATCH ("===") against string constants set by
LembarKerja.php (NOT stripos/str_contains, to avoid false positives from
user-supplied keterangan).
